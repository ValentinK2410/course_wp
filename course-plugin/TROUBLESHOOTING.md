# Решение проблем с плагином "Курсы Про"

## Проблема: Меню "Курсы Про" не появляется в боковой панели WordPress

### Решение 1: Проверка активации плагина

1. Перейдите в **Плагины → Установленные плагины**
2. Найдите плагин **"Курсы Про"**
3. Убедитесь, что он **активирован** (должна быть кнопка "Деактивировать")
4. Если плагин не активирован, нажмите **"Активировать"**

### Решение 2: Деактивация и повторная активация

1. Перейдите в **Плагины → Установленные плагины**
2. Нажмите **"Деактивировать"** для плагина "Курсы Про"
3. Подождите несколько секунд
4. Нажмите **"Активировать"** снова
5. Обновите страницу админ-панели (F5)

### Решение 3: Проверка ошибок PHP

1. Включите отображение ошибок в WordPress (если у вас есть доступ к wp-config.php):
   ```php
   define('WP_DEBUG', true);
   define('WP_DEBUG_LOG', true);
   define('WP_DEBUG_DISPLAY', false);
   ```

2. Проверьте файл `wp-content/debug.log` на наличие ошибок

3. Или проверьте консоль браузера (F12) на наличие JavaScript ошибок

### Решение 4: Проверка прав доступа

Убедитесь, что вы вошли как администратор сайта. Плагин требует прав администратора для отображения меню.

### Решение 5: Сброс правил перезаписи

1. Перейдите в **Настройки → Постоянные ссылки**
2. Нажмите кнопку **"Сохранить изменения"** (без изменения настроек)
3. Это сбросит правила перезаписи и перерегистрирует типы постов

### Решение 6: Проверка конфликтов с другими плагинами

1. Деактивируйте все другие плагины
2. Активируйте только "Курсы Про"
3. Проверьте, появилось ли меню
4. Если меню появилось, активируйте плагины по одному, чтобы найти конфликтующий

### Решение 7: Проверка файлов плагина

Убедитесь, что все файлы плагина находятся в правильной директории:

```
wp-content/plugins/course-plugin/
├── course-plugin.php
├── includes/
│   ├── class-course-post-type.php
│   ├── class-course-taxonomies.php
│   ├── class-course-admin.php
│   └── class-course-meta-boxes.php
```

### Решение 8: Проверка версии WordPress и PHP

- **WordPress:** версия 5.0 или выше
- **PHP:** версия 7.0 или выше

Проверить версию WordPress: **Инструменты → Здоровье сайта → Информация**

### Решение 9: Ручная проверка регистрации типа поста

Добавьте временно в файл `functions.php` вашей темы (или создайте тестовый плагин):

```php
add_action('init', function() {
    if (post_type_exists('course')) {
        echo 'Тип поста "course" зарегистрирован';
    } else {
        echo 'Тип поста "course" НЕ зарегистрирован';
    }
});
```

### Решение 10: Очистка кеша

Если используете плагины кеширования:
1. Очистите кеш плагина кеширования
2. Очистите кеш браузера (Ctrl+F5)
3. Попробуйте открыть админ-панель в режиме инкогнито

## Если ничего не помогло

1. Проверьте логи ошибок сервера
2. Создайте резервную копию сайта
3. Попробуйте переустановить плагин:
   - Удалите папку `wp-content/plugins/course-plugin/`
   - Загрузите плагин заново
   - Активируйте его
