# Инструкция по работе с Git и GitHub

## Основные команды Git

### 1. Проверка статуса репозитория

Проверяет, какие файлы были изменены, добавлены или удалены:

```bash
git status
```

### 2. Добавление файлов в индекс (staging area)

Добавить все измененные файлы:
```bash
git add .
```

Добавить конкретный файл:
```bash
git add путь/к/файлу.php
```

Добавить несколько файлов:
```bash
git add файл1.php файл2.php файл3.php
```

Добавить все файлы в определенной папке:
```bash
git add course-plugin/
```

### 3. Создание коммита (сохранение изменений)

Создать коммит с сообщением:
```bash
git commit -m "Описание изменений"
```

Примеры хороших сообщений коммитов:
```bash
git commit -m "Добавлен шаблон страницы отдельного курса"
git commit -m "Исправлена ошибка в фильтрах курсов"
git commit -m "Обновлены стили для мобильных устройств"
git commit -m "Добавлена функция дублирования курсов"
```

### 4. Отправка изменений на GitHub

Отправить изменения в удаленный репозиторий:
```bash
git push
```

Отправить изменения в конкретную ветку:
```bash
git push origin master
```

Отправить изменения и установить отслеживание ветки:
```bash
git push -u origin master
```

## Полный процесс отправки изменений на GitHub

### Шаг 1: Проверьте текущий статус
```bash
git status
```
Эта команда покажет:
- Какие файлы изменены (modified)
- Какие файлы добавлены (new file)
- Какие файлы удалены (deleted)
- Какие файлы не отслеживаются (untracked)

### Шаг 2: Добавьте файлы в индекс
```bash
# Добавить все изменения
git add .

# Или добавить конкретные файлы
git add course-plugin/templates/single-course.php
git add course-plugin/assets/css/frontend.css
```

### Шаг 3: Создайте коммит
```bash
git commit -m "Краткое описание того, что было изменено"
```

**Правила написания сообщений коммитов:**
- Используйте повелительное наклонение: "Добавить", "Исправить", "Обновить"
- Первая строка должна быть короткой (до 50 символов)
- Можно добавить подробное описание после пустой строки

Пример хорошего коммита:
```bash
git commit -m "Добавлен шаблон страницы отдельного курса

- Создан файл single-course.php с полной структурой страницы
- Добавлены стили для баннера, боковой панели и контента
- Реализована поддержка видео (YouTube, Vimeo)
- Добавлен блок с похожими курсами"
```

### Шаг 4: Отправьте изменения на GitHub
```bash
git push
```

Если это первый раз и нужно настроить отслеживание ветки:
```bash
git push -u origin master
```

## Дополнительные полезные команды

### Просмотр истории коммитов
```bash
# Краткий список последних коммитов
git log --oneline -10

# Подробный список коммитов
git log

# История с графиком веток
git log --oneline --graph --all
```

### Просмотр изменений в файлах
```bash
# Показать изменения в рабочей директории
git diff

# Показать изменения в индексе (staged)
git diff --staged

# Показать изменения конкретного файла
git diff путь/к/файлу.php
```

### Отмена изменений

Отменить изменения в файле (до добавления в индекс):
```bash
git checkout -- путь/к/файлу.php
```

Убрать файл из индекса (но сохранить изменения):
```bash
git reset HEAD путь/к/файлу.php
```

Отменить последний коммит (но сохранить изменения):
```bash
git reset --soft HEAD~1
```

### Работа с ветками

Посмотреть все ветки:
```bash
git branch
```

Создать новую ветку:
```bash
git branch название-ветки
```

Переключиться на ветку:
```bash
git checkout название-ветки
```

Создать и переключиться на новую ветку:
```bash
git checkout -b название-ветки
```

### Получение изменений с GitHub

Получить изменения с удаленного репозитория:
```bash
git pull
```

Получить изменения без слияния:
```bash
git fetch
```

## Типичные сценарии работы

### Сценарий 1: Вы внесли изменения в файлы и хотите отправить их на GitHub

```bash
# 1. Проверить статус
git status

# 2. Добавить все изменения
git add .

# 3. Создать коммит
git commit -m "Описание изменений"

# 4. Отправить на GitHub
git push
```

### Сценарий 2: Вы хотите отправить только определенные файлы

```bash
# 1. Добавить только нужные файлы
git add файл1.php файл2.php

# 2. Создать коммит
git commit -m "Обновлены только файл1 и файл2"

# 3. Отправить на GitHub
git push
```

### Сценарий 3: Вы хотите обновить локальные файлы с GitHub

```bash
# Получить и объединить изменения
git pull
```

### Сценарий 4: Вы случайно добавили файл в индекс и хотите его убрать

```bash
# Убрать файл из индекса
git reset HEAD путь/к/файлу.php

# Файл останется измененным, но не будет включен в коммит
```

## Настройка Git (если еще не настроено)

### Установка имени и email

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш@email.com"
```

### Проверка настроек

```bash
git config --list
```

## Решение проблем

### Ошибка: "Please tell me who you are"

Нужно настроить имя и email:
```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш@email.com"
```

### Ошибка: "Updates were rejected because the remote contains work"

Кто-то отправил изменения на GitHub раньше вас. Нужно сначала получить их:
```bash
git pull
# Разрешить конфликты, если они есть
git push
```

### Ошибка: "Permission denied"

Проверьте, что вы авторизованы в GitHub. Возможно, нужно настроить SSH ключи или использовать токен доступа.

## Быстрая шпаргалка

```bash
# Проверить статус
git status

# Добавить все изменения
git add .

# Создать коммит
git commit -m "Сообщение"

# Отправить на GitHub
git push

# Получить изменения с GitHub
git pull

# Посмотреть историю
git log --oneline -10
```

## Рекомендации

1. **Делайте коммиты часто** - лучше много маленьких коммитов, чем один большой
2. **Пишите понятные сообщения коммитов** - через месяц вы сами не вспомните, что означало "fix"
3. **Перед push проверяйте статус** - убедитесь, что отправляете то, что нужно
4. **Используйте .gitignore** - не коммитьте временные файлы, пароли, настройки IDE
5. **Делайте pull перед push** - если работаете в команде, сначала получите изменения
