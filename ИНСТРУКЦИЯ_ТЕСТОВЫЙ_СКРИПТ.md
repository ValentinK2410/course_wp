# Инструкция по использованию тестового скрипта синхронизации курсов

## Где должен находиться файл

Файл `test-course-sync.php` должен находиться в **КОРНЕ WordPress**, там же, где находится `wp-config.php`.

**НЕ в папке плагина!**

## Как правильно установить

### Вариант 1: Через FTP/SFTP

1. Скачайте файл `course-plugin/test-course-sync.php`
2. Загрузите его в корень WordPress (рядом с `wp-config.php`)
3. Откройте в браузере: `https://site.dekan.pro/test-course-sync.php`

### Вариант 2: Через SSH

```bash
# На сервере WordPress
cd /path/to/wordpress
# Скопируйте файл из плагина в корень
cp wp-content/plugins/course-plugin/test-course-sync.php ./
# Или создайте файл вручную, скопировав содержимое
```

## Структура файлов должна быть такой:

```
/var/www/wordpress/                    ← Корень WordPress
├── wp-config.php
├── wp-load.php
├── test-course-sync.php              ← Файл должен быть ЗДЕСЬ
├── wp-content/
│   └── plugins/
│       └── course-plugin/
│           └── test-course-sync.php  ← НЕ здесь!
```

## Использование

1. Откройте в браузере: `https://site.dekan.pro/test-course-sync.php`
2. Скрипт покажет:
   - Настройки Laravel API
   - Список курсов в WordPress
   - Результат тестового запроса к Laravel
   - Последние записи в логах

## Безопасность

**ВАЖНО:** После использования **ОБЯЗАТЕЛЬНО удалите файл** `test-course-sync.php` из корня WordPress!

Файл проверяет права администратора, но лучше не оставлять его на сервере.

## Альтернатива: Использование через WP-CLI

Если у вас есть доступ к WP-CLI, можете выполнить проверку через командную строку:

```bash
cd /path/to/wordpress
wp eval-file wp-content/plugins/course-plugin/test-course-sync.php
```

Но для этого нужно будет немного изменить скрипт, убрав HTML вывод.
