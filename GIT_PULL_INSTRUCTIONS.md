# Краткая инструкция: Как получить изменения с GitHub

## Основная команда

```bash
git pull
```

Эта команда:
1. Получает изменения с GitHub
2. Автоматически объединяет их с вашими локальными файлами

## Пошаговая инструкция

### Шаг 1: Откройте терминал в папке проекта

```bash
cd /Users/valentink2410/PhpstormProjects/course_wp
```

### Шаг 2: Получите изменения

```bash
git pull
```

## Альтернативный способ (две команды)

Если хотите сначала посмотреть, что изменилось:

```bash
# 1. Получить изменения без объединения
git fetch

# 2. Посмотреть, что изменилось
git log HEAD..origin/master

# 3. Объединить изменения
git pull
```

## Если возникли конфликты

Если Git не может автоматически объединить изменения, вы увидите сообщение о конфликтах.

**Решение:**
1. Откройте файлы с конфликтами
2. Найдите маркеры конфликтов: `<<<<<<<`, `=======`, `>>>>>>>`
3. Выберите нужный код или объедините изменения
4. Сохраните файлы
5. Выполните:
```bash
git add .
git commit -m "Разрешение конфликтов"
```

## Проверка статуса перед pull

Перед получением изменений можно проверить статус:

```bash
# Проверить, есть ли незакоммиченные изменения
git status

# Если есть изменения, можно их сохранить
git add .
git commit -m "Сохранение локальных изменений"

# Затем получить изменения с GitHub
git pull
```

## Быстрая шпаргалка

```bash
# Получить изменения с GitHub
git pull

# Проверить статус
git status

# Посмотреть последние коммиты
git log --oneline -5
```

## Частые вопросы

**Q: Что делать, если я получил ошибку "Your local changes would be overwritten"?**

A: У вас есть незакоммиченные изменения. Сначала сохраните их:
```bash
git add .
git commit -m "Сохранение изменений"
git pull
```

**Q: Как получить изменения из конкретной ветки?**

A: Укажите название ветки:
```bash
git pull origin название-ветки
```

**Q: Как отменить pull, если что-то пошло не так?**

A: Вернуться к предыдущему состоянию:
```bash
git reset --hard HEAD~1
```
